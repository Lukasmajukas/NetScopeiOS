# CLAUDE.md — NetScope for iPhone

Guidance for AI agents working in this repo. Pair with `ARCHITECTURE.md` (module detail)
and `README.md` (product overview).

## What it is
A native SwiftUI iPhone app: internet **speed test** (history + Ookla-format CSV export),
**Bonjour/mDNS device finder**, live **connection** info (Wi-Fi/cellular), and Wi-Fi
**Learn** explainers. A **Pro** tier (preview toggle in Settings) adds a coverage heatmap
and ISP enrichment. It is deliberately **not** a port of the macOS NetScope
(`../NetScopeMac/`) — iOS forbids the LAN scanning / shell tooling that one relies on.

## Build & run
This project is **generated by XcodeGen** from `project.yml`; the `.xcodeproj` is an artifact.

```bash
brew install xcodegen            # one time
cd NetScopeiOS
xcodegen generate                # REQUIRED after adding/moving/renaming source files
open NetScope.xcodeproj
```

Then set your signing **Team** (Signing & Capabilities) and Run on a device or simulator.
Targets iOS 16+.

> ⚠️ If you add or move a `.swift` file and skip `xcodegen generate`, Xcode builds the
> *old* file list — you'll get missing-file / missing-symbol errors. Always regenerate.

```
  Speed/    SpeedTest.swift       model, persistence (HistoryStore), Cloudflare +
                                  M-Lab/NDT7 engine, Ookla CSV export
            SpeedTestView.swift   animated gauge + speed UI
  Devices/  Devices.swift         Bonjour/mDNS DeviceBrowser
  Connection/ Connection.swift    live Wi-Fi/cellular state, public IP/carrier, map
  Learn/    Learn.swift           band/width/speed explainers (static content)
  Settings/ Settings.swift        settings, history mgmt, Pro toggle, enrichment config
            Privacy.swift         in-app Privacy Policy + Terms views
  Pro/      Pro.swift             ProManager gate — every view reads `pro.isPro`
            Coverage.swift        speed-tile heatmap + optional Mac "backbone" sync
            Enrichment.swift      Ookla SubscriberServiceResponse model + sample/real client
Core/       Theme.swift           colour palette / styling
            Tools.swift           network diagnostics (latency, DNS, TCP, IP detail)
Assets.xcassets, Info.plist       stay at NetScope/ root (referenced by project.yml)
```

## Conventions & key facts
- **State**: `@Observable` model classes injected via the SwiftUI environment
  (`HistoryStore`, `LocationProvider`, `ProManager`); views read them with `@Environment(...)`.
- **Pro gating** is centralised in `ProManager` (`Features/Pro/Pro.swift`). Today it's a
  preview switch; to make it a real purchase, replace the stored flag with a
  `Transaction.currentEntitlements` (StoreKit 2) check — no feature code changes.
- **Speed engine**: Cloudflare edge by default; optional **M-Lab/NDT7** cities. M-Lab
  publishes results (including your IP) as open data, so the first M-Lab run shows a
  one-time consent prompt. History persists locally; export matches Ookla CSV byte-for-byte.
- **iOS limits shape the app**: no ARP/ping sweeps, no MAC addresses, no Wi-Fi scanning API.
  The Devices tab is Bonjour-only and auto-hidden on cellular.
- `Info.plist` is committed (`GENERATE_INFOPLIST_FILE: NO`) and holds the usage strings
  (Local Network, Location) — edit it there, not in target settings.

## Gotchas
- Don't move `Info.plist` or `Assets.xcassets` out of `NetScope/` without updating
  `project.yml` (`INFOPLIST_FILE: NetScope/Info.plist`).
- `build/` and `.auto-deploy.*` are gitignored local artifacts — never commit them.
- Swift files in subfolders don't need import changes: all files in the target share one
  module. The folders are organisation only; XcodeGen picks them up recursively.
